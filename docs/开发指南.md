## 2. 开发策略（重要）

### 2.1 原则

1. **先后端协议，后客户端**：先把 REST + WSS 跑通，Flutter 再对接。
2. **E2EE 分两步**：先明文 + TLS 打通全链路，再替换为客户端加密（降低调试难度）。
3. **每个阶段可演示**：每 1–2 周有一个可点的里程碑。
4. **真机尽早测**：WebSocket、文件、Secure Storage 在模拟器上易有差异。

### 2.2 推荐顺序（一个人开发）

```
Week 1–2   后端 Auth（含邮箱找回）+ User + Docker 本地环境
Week 2–3   会话/消息 REST + WSS（暂明文）
Week 3–4   Flutter：登录/注册/找回密码 + 会话列表 + 聊天（明文）
Week 5–6   单聊 E2EE + 文件上传
Week 7–8   群聊 Epoch/GMK + 群公告
Week 9–10  多设备链接 + device_sync
Week 11    境外 VPS 部署 + HTTPS
Week 12    测试、修 bug、内测分发
```

若 2 人：**一人后端 + 加密，一人 Flutter**，Week 3 起并行。

## 4. 阶段 1：账号体系（含邮箱找回密码）

### 4.1 后端任务清单

- [ ] `users` 表：`email` UNIQUE、`username` UNIQUE、`password_hash`、`identity_public_key`
- [ ] `user_devices` 表
- [ ] `password_reset_tokens` 表
- [ ] `POST /api/auth/register`
- [ ] `POST /api/auth/login`（body: email, password, device_id, device_name）
- [ ] `POST /api/auth/refresh`
- [ ] `POST /api/auth/forgot-password`
- [ ] `POST /api/auth/reset-password`
- [ ] JWT 中间件
- [ ] 登录限流（如 5 次/分钟/IP）

### 4.2 找回密码实现要点

```
forgot-password:
  1. 查 email，不存在也返回 200（防枚举邮箱）
  2. 生成 32 字节 random token → SHA256 存库
  3. expires_at = now + 30min
  4. 发邮件：链接或 6 位验证码（二选一，建议 App Deep Link + token）

reset-password:
  1. 校验 token_hash、未过期、used_at IS NULL
  2. 更新 password_hash
  3. 标记 token 已用
  4. DELETE 该用户所有 refresh_token
```

**邮件模块 `internal/mail`：**

```go
// 开发环境：MAIL_DRIVER=log 只打印到控制台
// 生产环境：MAIL_DRIVER=smtp 或 resend
func SendPasswordReset(to, resetURL string) error
```

环境变量示例：

```env
MAIL_FROM=noreply@yourdomain.com
SMTP_HOST=smtp.resend.com
SMTP_PORT=587
SMTP_USER=resend
SMTP_PASS=re_xxx
APP_PUBLIC_URL=https://im.example.com
```

### 4.3 Flutter 任务清单

- [ ] `lib/config/env.dart`：dev/prod 基地址
- [ ] `AuthService`：register、login、refresh
- [ ] 页面：`LoginScreen`、`RegisterScreen`、`ForgotPasswordScreen`、`ResetPasswordScreen`
- [ ] 首次启动生成 `device_id` 写入 `flutter_secure_storage`
- [ ] 注册时在客户端生成 Identity Key Pair，注册请求带 `identity_public_key`

### 4.4 本阶段验收

- 两台模拟器/真机注册不同账号
- 忘记密码 → 收邮件（或 dev 日志里的链接）→ 重置成功
- 旧 Token 调 API 返回 401

## 6. 阶段 3：单聊 E2EE

### 6.1 加密库选型（Flutter）

| 方案 | 说明 |
|------|------|
| **libsignal_protocol_dart** 或 **cryptography** + 自实现 ratchet | 标准 E2EE |
| 简化版（100 人可接受） | X25519 ECDH + AES-GCM 会话密钥，不做完整 Double Ratchet |

建议：**第一期用 `cryptography` 包实现 X25519 + AES-GCM**；单聊 session 状态存 secure storage。

### 6.2 客户端 `lib/crypto/`

```
identity.dart      生成/加载 IK
dm_session.dart    与每个用户维护 session key
encrypt.dart       encrypt(plaintext, key) / decrypt
```

流程：

1. 注册时生成 IK，上传公钥
2. 首次单聊：拉对方公钥 → ECDH → session key
3. `send` 前加密，`content` 字段改为 `ciphertext` Base64
4. 服务端逻辑 **不变**（仍只存字符串）

### 6.3 本阶段验收

- 数据库里 `messages.content` 为乱码
- 抓包看不到明文
- A/B 正常聊天

## 12. 测试建议

### 12.1 运行 Go 测试

在 **`backend` 目录**执行（与 `go run ./cmd/server` 相同）：

```powershell
cd D:\施玮书房\IHope\backend
go test ./... -count=1 -v
```

| 命令 | 说明 |
|------|------|
| `go test ./...` | 跑全部包 |
| `go test ./... -v` | 打印每个测试名 |
| `go test ./... -count=1` | 禁用缓存，强制重跑 |
| `go test ./internal/server/... -v -count=1` | 只跑 HTTP 集成测试 |
| `go test ./internal/auth/... ./internal/jwt/... -v` | 只跑单元测试（无需数据库） |

**前置：** Docker PostgreSQL 已启动（见 [Windows开发环境.md](./Windows开发环境.md)）。集成测试会自动读 `deploy/.env`（含 `DB_PORT=5433`），并执行 migration。

也可显式指定（任意工作目录均有效）：

```powershell
$env:TEST_DATABASE_URL = "postgres://ihope:devpassword@127.0.0.1:5433/ihope?sslmode=disable"
go test ./internal/server/... -v -count=1
```

### 12.2 输出说明

| 结果 | 含义 |
|------|------|
| `PASS` / `ok` | 通过 |
| `SKIP: database not available` | 连不上库（容器未起、端口/密码不对） |
| `[no test files]` | 该包无 `*_test.go`，正常 |

集成测试（`internal/server`）覆盖：注册登录、找回/修改密码、会话与消息 REST。

### 12.3 其他测试

| 类型 | 做法 |
|------|------|
| API（手工） | Postman，见 [API与Postman测试指南.md](./API与Postman测试指南.md) |
| WSS | Postman WebSocket 或 Go 集成测试 |
| E2EE | 固定 test vectors：加密后再解密等于原文 |
| 多设备 | 两个模拟器 + 两个账号，列 checklist 人工测 |
| 找回密码 | 测过期 token、重复使用 token、错误邮箱 |

**必测场景清单：**

1. 注册 → 登录 → 单聊 → 杀进程重进  
2. 忘记密码 → 重置 → 旧设备被踢  
3. 建群 → 发公告 → 拉新人 → 新人无历史  
4. 链接第二台设备 → 内容一致

## 13. 常见问题

### Q：找回密码后能看到聊天记录吗？

**不能自动恢复。** 密码只管「账号门禁」；E2EE 消息密钥在设备或加密备份里。  
重置密码后需：**链接旧设备** 或 **输入云备份口令**。UI 必须在重置成功页说明这一点。

### Q：应该先写 Flutter 还是先写 Go？

**先 Go。** 没有 API，App 只能 mock，后期联调返工多。

### Q：E2EE 能不能第一期就上？

可以，但建议 **阶段 2 用明文打通 WSS**，阶段 3 再换加密；否则网络/推送/分页 bug 与加密 bug 缠在一起难查。

### Q：一个人做不完怎么办？

砍优先级：

1. P0：邮箱登录 + 单聊 E2EE + 群聊（无历史）  
2. P1：多设备链接  
3. P2：文件、推送、云备份
