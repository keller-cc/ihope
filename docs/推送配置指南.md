# 推送配置指南（国内 + 海外）

## 能不能直接用？

| 场景 | 能否收到系统横幅 |
|------|------------------|
| **零配置 + 用户开通知** | **能**（App 在后台、进程仍存活时，走 WebSocket + 本地通知，**无需极光/FCM**） |
| **App 被系统完全杀掉** | **不能**（需配置极光/FCM 离线推送） |
| **零配置且未开通知** | **不能** |

类似 QQ 的分层：

1. **前台聊天**：WebSocket 实时，当前会话内不弹系统横幅  
2. **后台但进程存活**：前台服务保 WebSocket + **本机本地通知**（不依赖第三方）  
3. **进程被系统杀掉**：极光（国内）/ FCM（海外）离线推送  

---

## 架构简述

```
Tom 发消息 → Go 后端
              ├─ 设备 WebSocket 在线 → 仅 WS 下发（不发极光/FCM）
              └─ 设备离线 → 按 platform 发推送
                    ├─ android_cn → 极光
                    └─ android    → FCM

Bob 手机：
  后台存活 → WS 收到密文 → 本机解密 → 系统栏显示明文
  已被杀   → 极光/FCM 仅传密文 + 元数据 → 本机解密 → 系统栏显示明文
```

**安全模型**：第三方推送通道（极光/FCM）与服务器之间传递的是 **消息密文**（`ciphertext`）及会话 ID 等元数据，**不含消息明文**。系统通知栏上用户看到的文字，一律在 **本机用 E2EE 密钥解密后** 由 `flutter_local_notifications` 展示；解密失败时仅显示 `[图片]`、`收到一条新消息` 等占位，不展示密文。

- 用户须在 **个人资料 → 通知** 中开启，并授予系统通知权限  
- 国内机型建议允许 **自启动 / 后台运行**（否则进程易被系统杀掉，只能依赖极光）

## 一、后端（`deploy/.env`）

**仅测「后台长连接 + 本地通知」时可不配任何密钥**，后端对在线设备不会发第三方推送。

```env
# 可选。仅 App 被杀后的离线兜底需要
JPUSH_APP_KEY=
JPUSH_MASTER_SECRET=
FCM_SERVER_KEY=
```

### 国内（极光，离线兜底）

1. 打开 [极光开发者服务](https://www.jiguang.cn/) 注册（有免费额度）
2. 控制台 → 创建应用 → 选择 **Android**
3. 包名填：**`com.clprince.ihope.cn`**（国内 flavor 的 applicationId）
4. 记录 **AppKey**、**Master Secret**
5. 写入 `.env`：

```env
JPUSH_APP_KEY=你的AppKey
JPUSH_MASTER_SECRET=你的MasterSecret
```

### 海外（Firebase / FCM，离线兜底）

1. [Firebase Console](https://console.firebase.google.com/) 创建项目
2. 添加 Android 应用，包名：**`com.clprince.ihope`**
3. 下载 `google-services.json` → 放到 `mobile/android/app/`
4. 项目设置 → 云消息传递 → **服务器密钥（Legacy）** → 写入：

```env
FCM_SERVER_KEY=你的服务器密钥
```

> 生产环境建议迁移 FCM HTTP v1；当前实现为 Legacy API，适合小团队起步。

## 二、Android 客户端

项目有两个 **productFlavor**：

| Flavor | 包名 | 离线兜底 | 编译 define |
|--------|------|----------|-------------|
| `domestic` | `com.clprince.ihope.cn` | 极光 | `PUSH_CHANNEL=jpush` |
| `global` | `com.clprince.ihope` | FCM | `PUSH_CHANNEL=fcm` |

**后台本地通知**两个 flavor 均可用，不依赖上表中的第三方配置。

### 国内包

1. 极光控制台包名必须与 **`com.clprince.ihope.cn`** 一致（仅离线兜底需要）
2. 在 `mobile/android/local.properties` 增加（勿提交 git）：

```properties
JPUSH_APPKEY=与极光控制台相同的AppKey
```

3. 运行：

```powershell
cd D:\IHope\mobile
flutter pub get
flutter run --flavor domestic --dart-define-from-file=config/domestic.json
```

### 海外包

1. 确保 `mobile/android/app/google-services.json` 存在（仅离线兜底需要）
2. 运行：

```powershell
flutter run --flavor global --dart-define-from-file=config/global.json
```

### 未指定 flavor 时

Android 现在**必须**带 `--flavor domestic` 或 `--flavor global`，否则 Gradle 会报错。

## 三、用户侧（真机）

1. 登录 App  
2. **个人资料 → 通知 → 开启「后台新消息通知」**  
3. 允许系统通知权限  
4. **国内机型**：建议在系统设置中允许 IHope **自启动 / 后台运行 / 锁屏显示**  

开启后切到后台，状态栏会出现 **「IHope 正在接收新消息」** 常驻通知（保 WebSocket），新消息会弹独立横幅。

## 四、验证

### 长连接 + 本地通知（零第三方配置）

1. 后端可不设 `JPUSH_*` / `FCM_SERVER_KEY`  
2. Bob 登录并 **开启通知**  
3. Bob **按 Home 切到后台**（不要划掉 App）  
4. Tom 发消息 → Bob 应看到系统横幅  
5. 后端日志 **不应** 对该在线设备发 `[push]`（WS 在线则跳过）

### 离线兜底（极光/FCM）

1. 配齐后端密钥 + 客户端 flavor + 用户开启通知  
2. Bob **划掉 App 或强制停止**  
3. Tom 发消息 → Bob 应看到系统横幅（走极光/FCM）  
4. 点击横幅 → 进入对应会话

## 五、常见问题

**Q：不配置极光/FCM 能收通知吗？**  
A：能。只要 App 进程还在后台、WebSocket 连着，就走本地通知。被系统杀掉后才需要极光/FCM。

**Q：App 被杀掉还能推吗？**  
A：可以，但需配置极光/FCM。若用户「强制停止」或关闭通知权限，会收不到。

**Q：国内能用 Firebase 吗？**  
A：不可靠。国内请用 **domestic + 极光** 作离线兜底。

**Q：iOS？**  
A：无 Android 式前台服务；后台短暂存活时可走本地通知，长期离线依赖 APNs/FCM。

**Q：推送内容安全吗？**  
A：极光/FCM 链路只传 **密文**；通知栏明文由手机本地解密后展示。解密失败时显示类型占位（如 `[图片]`），不展示密文。
