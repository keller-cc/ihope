# 无备案国内用户部署指南

面向 **主要用户在中国大陆**、**不想做 ICP 备案** 的个人或小团队内测方案。

核心思路：**服务器放在境外（推荐香港）+ 自有域名 + 源站 HTTPS**。域名可以购买，但 **服务器不在大陆** 则 **不需要** 大陆 ICP 备案。

> 正式大规模商用 IM 可能另有资质要求；本文仅说明 **技术部署**，不构成法律意见。

---

## 架构一览

```
国内用户 App（domestic 包 + 极光）
        │
        │  https://im.你的域名.com
        │  wss://im.你的域名.com/ws
        ▼
香港 VPS（Docker：postgres + backend + nginx）
```

| 组件 | 建议 |
|------|------|
| **VPS 地域** | **香港** 优先（离大陆近）；备选新加坡、东京 |
| **域名** | `.com` 等均可；在任意注册商购买 |
| **DNS** | **A 记录** 直连 VPS IP；**不要**开 Cloudflare **橙云**（国内慢） |
| **HTTPS** | VPS 上 certbot + `nginx-ssl.conf` |
| **App** | `--flavor domestic` + `prod.json` 填 HTTPS 域名 |
| **推送** | 极光（见 [推送配置指南.md](./推送配置指南.md)） |
| **大文件** | IM 最大 300MB；更大走 **1t1 网盘**（App 内已有入口） |

---

## 与「备案方案」的区别

| | 无备案（本文） | 大陆云 + 备案 |
|--|----------------|---------------|
| 服务器 | 香港 / 境外 | 阿里云、腾讯云等大陆节点 |
| ICP 备案 | **不需要** | 需要（约 2～4 周） |
| 国内访问速度 | 一般够用，视线路 | 通常更快、更稳 |
| Cloudflare 橙云 | **不推荐** | 国内也不推荐当主路径 |

## 步骤 1：购买域名

1. 在 **Namecheap、Cloudflare Registrar、阿里云万网** 等任选一家注册域名（例如 `yourname.com`）。
2. 完成域名 **实名**（注册商要求）。
3. 在 DNS 控制台添加记录（先拿到 VPS IP 后再填也可）：
   - 类型：**A**
   - 主机记录：`im`（则访问地址为 `im.yourname.com`）
   - 记录值：**香港 VPS 公网 IP**
   - **不要**开启 Cloudflare 橙云代理；若域名在 Cloudflare，请设为 **仅 DNS（灰云）**。

---

## 步骤 2：租用香港 VPS

推荐配置（内测起步）：

- **系统**：Ubuntu 22.04 LTS  
- **规格**：1 核 2GB 内存、30GB+ 磁盘  
- **开放端口**：22（SSH）、80、443  

常见厂商：阿里云香港、腾讯云香港、Vultr、DigitalOcean、Bandwagon 等。

SSH 登录后安装 Docker：

```bash
sudo apt update && sudo apt install -y docker.io docker-compose-plugin git
sudo usermod -aG docker $USER
# 重新登录 SSH 使 docker 组生效
```

---

## 步骤 3：部署 IHope

```bash
git clone <你的仓库> /opt/ihope
cd /opt/ihope/deploy
cp ../.env.example .env
```

编辑 `.env`（至少修改以下项）：

```env
DB_PASSWORD=强密码
JWT_SECRET=至少32字符的随机串
ADMIN_SECRET=管理后台密钥

APP_PUBLIC_URL=https://im.yourname.com
CORS_ALLOW_ORIGIN=https://im.yourname.com

# 国内推送（可选，不配则后端 log 模式）
PUSH_DRIVER=jpush
JPUSH_APP_KEY=你的极光AppKey
JPUSH_MASTER_SECRET=你的极光MasterSecret

CLOUD_DRIVE_URL=https://1t1.org
SERVER_VERSION=2026-07-03 0.1.0 version
```

启动：

```bash
docker compose up -d --build
curl http://127.0.0.1/api/health
```

确认 DNS 已生效：`ping im.yourname.com` 应指向 VPS IP。

---

## 步骤 4：启用 HTTPS（certbot）

1. 复制 SSL 模板并改域名：

```bash
cd /opt/ihope/deploy
cp nginx-ssl.conf.example nginx-ssl.conf
# 将 im.example.com 全部改为 im.yourname.com
```

2. 申请证书（需暂时占用 80 端口）：

```bash
sudo apt install certbot
docker compose stop nginx
sudo certbot certonly --standalone -d im.yourname.com
docker compose start nginx
```

3. 修改 `docker-compose.yml` 中 nginx 服务，挂载 SSL 配置与证书（示例见 [deploy/README.md](../deploy/README.md)）：

```yaml
nginx:
  ports:
    - "80:80"
    - "443:443"
  volumes:
    - ./nginx-ssl.conf:/etc/nginx/conf.d/default.conf:ro
    - /etc/letsencrypt:/etc/letsencrypt:ro
```

4. 重载：

```bash
docker compose up -d nginx
curl https://im.yourname.com/api/health
```

证书续期：`certbot renew` 后重载 nginx。

---

## 步骤 5：打国内生产 APK

在 **开发机 Windows** 上：

```powershell
cd D:\IHope\mobile
copy config\prod.json.example config\prod.json
# 编辑 prod.json：
# { "API_BASE": "https://im.yourname.com" }

# 极光 AppKey（可选，见 mobile/android/local.properties）
.\scripts\build-release.ps1 -Flavor domestic
```

产物：`build\app\outputs\flutter-apk\app-domestic-release.apk`

**内测分发方式（任选）：**

- 直接发 APK 文件  
- 上传到 VPS：`docker cp app-domestic-release.apk ihope-backend:/data/uploads/releases/latest.apk`  
- App 内「检查版本」可下载（需 `APP_PUBLIC_URL` 与 health 配置正确）

用户仍可在 App **个人资料 → 服务器** 修改地址；生产环境建议固定 HTTPS 域名。

---

## 步骤 6：极光推送（可选）

1. [极光控制台](https://www.jiguang.cn/) 创建应用，包名 **`com.clprince.ihope.cn`**（domestic flavor）。  
2. `deploy/.env` 填 `JPUSH_APP_KEY`、`JPUSH_MASTER_SECRET`，`PUSH_DRIVER=jpush`。  
3. `mobile/android/local.properties` 增加 `JPUSH_APPKEY=...`（勿提交 Git）。  
4. 重启 backend；真机安装 domestic 包，登录后在 **通知设置** 打开权限。

未配置密钥时：App 与聊天正常，仅无离线系统横幅；后端 `PUSH_DRIVER=log` 可开发调试。

详见 [推送配置指南.md](./推送配置指南.md)。

## 常见问题

### 没有域名可以吗？

仅 **自己用 IP 测 docker** 可以；给其他人长期使用 **强烈建议买域名 + HTTPS**，否则证书、邮件链接、换 VPS 都很麻烦。

### 为什么不用 Cloudflare 橙云？

国内用户经橙云往往 **更慢**，WebSocket 聊天更容易断；且橙云有 **~100MB 上传上限**。无备案方案用 **DNS 直连香港 VPS** 即可。若仍想用 Cloudflare，见 [Cloudflare部署指南.md](./Cloudflare部署指南.md)（**灰云** 或仅 DNS）。

### 访问慢或偶尔连不上？

境外 VPS 受运营商国际线路影响，属正常现象。内测可接受；用户增多后再评估是否 **大陆云 + 备案**。

### 和开发环境的关系

本地开发不变：`docker-compose.dev.yml` + `go run` + `flutter run`。本文仅 **生产/内测** 路径。
